CHAPTER ONE
INTRODUCTION

1.1 Background Information
University course timetabling is the process of assigning courses, lecturers, rooms, and time slots in a way that meets all required rules and respects preferences as much as possible. The required rules (hard constraints) include no lecturer teaching two classes at the same time, no student group having two classes in the same slot, and rooms being large enough for the number of students. The preferred rules (soft constraints) include lecturers teaching at their chosen times, avoiding long gaps between classes for students, and keeping related courses on the same day where possible. University course timetabling is recognized as an NP-hard optimization problem, which means that finding the best possible solution becomes extremely difficult as the number of courses and constraints increases (Chen et al., 2021). 

Because of this difficulty, researchers and developers use heuristic and meta-heuristic methods that produce good-quality timetables within acceptable time limits. Common methods include genetic algorithms, simulated annealing, and tabu search (Abdipoor et al., 2023). These approaches have been applied successfully to real university datasets and have reduced clashes and improved room utilization compared with manual schedules (Song et al., 2018; Rezaeipanah et al., 2020). As universities have grown in size and complexity, manual methods have become inadequate.

However, most existing timetabling systems require input data to be provided in structured formats such as spreadsheets or database files. This requirement creates a barrier for non-technical users, including departmental administrators and academic planning officers who may not have the necessary skills to prepare the data correctly. Recent advances in natural language processing (NLP) offer a solution to this problem. Specifically, rule-based NLP techniques and pattern matching enable systems to accept requirements written in ordinary English and automatically extract the relevant constraints (Aa et al., 2019). This approach reduces preparation time and makes scheduling tools accessible to staff without programming knowledge.

Hyper-heuristics provide an additional improvement by managing several low-level heuristics and selecting or combining them during the search process. This approach increases flexibility and often produces better results than using a single method for all instances (Soria-Alcaraz et al., 2015). Recent work has shown that hyper-heuristics perform well on standard timetabling benchmarks and on real university data (Iqbal et al., 2021).

The present study builds on these developments by creating a system that accepts natural language input, converts it into formal constraints using rule-based NLP methods, and then applies multiple algorithms within a hyper-heuristic framework (genetic algorithm, simulated annealing, and tabu search) to generate timetables. The system will compare the performance of the algorithms on the same set of constraints and display the results to the user. This combination addresses the need for an accessible, effective, and comparative timetabling tool suitable for use in Covenant University and similar institutions.

1.2 Statement of the Problem
Most existing university timetabling systems require users to enter data in fixed formats, such as Excel spreadsheets or database files. This creates a major difficulty for many school staff who handle scheduling but do not have programming or technical skills. When non-technical users try to prepare the data themselves, they often make mistakes that lead to incorrect timetables. As a result, administrators spend extra time fixing errors manually, and the final schedule still contains clashes or poor room assignments (Lalitha et al., 2023). These problems reduce efficiency and cause frustration.

Another important limitation is that no single optimization algorithm performs best for every type of timetable or set of constraints. Without testing different algorithms on the actual constraints of a particular semester, it is difficult to know which one will give the best result (Soria-Alcaraz et al., 2015). Current systems usually offer only one fixed algorithm, so users cannot see or choose the most suitable method for their specific case.

In addition, very few timetabling tools can accept requirements written in ordinary English. While NLP can successfully extract rules from plain text (Zhong et al., 2019), most practical scheduling systems lack this feature. When natural language input is missing, non-technical staff remain dependent on technical experts to enter the constraints correctly. Finally, there is limited research that combines natural language input, multiple hyper-heuristic algorithms, and a clear comparison of their performance in a single web-based application (Muklason et al., 2024).

1.3 Aim and Objectives of the Study
The aim of this work is to integrate natural language reasoning into hyper-heuristic algorithms to automate and simplify university course scheduling. To achieve this aim, this project work will follow these specific objectives:
1. To construct a structured dataset suitable for integrating natural language reasoning into hyper-heuristic algorithms.
2. To use rule-based Natural Language Processing (NLP) tools to process hard and soft constraints from plain English into a readable input for optimization algorithms.
3. To evaluate and compare the different meta-heuristic algorithms (Genetic Algorithm, Simulated Annealing, and Tabu Search).
4. To demonstrate the practical value of the best-performing algorithm using a web application.

1.4 Methodology
The methodologies adopted for this study are:
1. Construction of a structured dataset from Covenant University.
2. Setting up a rule-based NLP parser using regular expressions to process user prompts into structured JSON constraints.
3. Implementation and comparative evaluation of three optimization algorithms: Genetic Algorithm, Simulated Annealing, and Tabu Search.
4. Development of a full-stack web application using Streamlit for the user interface, FastAPI for the backend, and SQLite for database management.

1.5 Significance of the Study
1.5.1 Practical Significance
The system allows scheduling staff to enter requirements in everyday English instead of complex file formats. This removes the technical barrier that prevents many administrators from using automated timetabling tools. By comparing three algorithms on the same constraints, universities can automatically choose the strongest algorithm, leading to fewer clashes and faster preparation. The Streamlit web application provides a ready-to-use system.

1.5.2 Academic and Research Significance
The work provides fresh empirical results on the performance of optimization algorithms when their input constraints come from natural language processing rather than manually structured data, a sparsely explored area (Aa et al., 2019; Chen et al., 2021). It presents a complete framework combining NLP parsers with hyper-heuristic solvers. The findings contribute to making advanced artificial intelligence techniques accessible to non-expert users.
